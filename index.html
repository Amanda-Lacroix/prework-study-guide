Prework Study Guide

HTML

  • The head element (Metadata used by the browser; not seen by users) contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML Elements are case-sensitive.
  • HTML Atributes define additional information about an element and always defined in the beginning element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.
  • Inline CSS apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • The most common way to style a webpage is by using an external CSS style sheet. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • A link element is required in the HTML's head in order for the sytle sheet to apply.

GIT

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A : stages the modifications and indicates we want to add all changes
  • git commit -m : commits the changes with a commit message
  • git pull: receives a branch's modifications into the local environment as defined by user. EX. git pull origin main : from the origin (Git Repo) from the branch (main). This ensures you have the most up-to-date code before making changes.
  • git push: pushes the changes we have locally to our remote GitHub branch as defined by user. EX git push origin feature/starter-code
  • Pull Request is used to merge a coparison branch (feature branch) into the base branch(main)
